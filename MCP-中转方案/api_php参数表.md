以下是代码中核心参数的详细介绍表


| **参数分类** | **参数名**   | **类型** | **含义说明**                                                                 | **取值范围/示例**                          | **核心作用**                                  |
|--------------|--------------|----------|------------------------------------------------------------------------------|-------------------------------------------|-----------------------------------------------|
| **请求参数** | `question`   | string   | 用户提交的问题内容，为问答交互的输入源，必填项（新请求时）。                   | "从北京到上海的火车票有哪些？"             | 作为大模型的输入，触发问答处理流程。          |
| **请求参数** | `questionId` | string   | 问题唯一标识，由服务端生成（新请求时返回），轮询时需携带，非必填（新请求时为空）。 | "qid_66f2d8a3b7c8d1234567890"             | 关联客户端轮询请求与服务端的处理结果。        |
| **响应参数** | `questionId` | string   | 服务端返回的问题唯一标识，新请求时生成，轮询时回传原标识。                     | "qid_66f2d8a3b7c8d1234567890"             | 供客户端后续轮询查询结果使用。                |
| **响应参数** | `answer`     | string   | 服务端返回的结果内容，可能是中间状态提示、错误信息或最终答案。                 | "正在处理，请稍候..."<br>"G101次列车 08:00发车" | 向客户端展示问题的处理进度或最终结果。        |
| **响应参数** | `verify`     | string   | 结果校验标记，用于判断`answer`是否为大模型处理完成的最终结果。                 | 空字符串（未完成）<br>"<!--COMPLETE-->"（完成） | 客户端通过此标记决定是否停止轮询（完成时停止）。 |


### 补充说明：
1. **参数交互逻辑**：
   - 新请求：客户端仅传递`question`，服务端返回`questionId`（新生成）、`answer`（临时提示）、`verify`（空）。
   - 轮询请求：客户端传递`questionId`，服务端返回相同`questionId`、`answer`（当前结果）、`verify`（状态标记）。

2. **特殊场景处理**：
   - 若`question`为空（新请求）：`answer`返回“错误: 缺少问题内容”，`questionId`和`verify`为空。
   - 若`questionId`无效（轮询）：`answer`返回“问题记录不存在或已过期”，`verify`为空。

3. **数据格式**：
   - 所有参数均通过JSON格式传输，字符编码为UTF-8。
   - `verify`的完成标记`<!--COMPLETE-->`为固定字符串，不可修改（服务端自动追加到最终`answer`中）。
